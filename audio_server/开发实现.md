WebRTC 语音通信跨协议信令服务器项目简介
一、项目核心定位
本项目是面向 Web 浏览器与 ESP32 嵌入式设备的 WebRTC 语音通信解决方案，核心通过 MQTT 客户端与 WebSocket 信令服务器的协议桥接能力，解决 Web 端（WebSocket 协议）与 ESP32 端（MQTT 协议）的协议异构问题，实现两端 WebRTC 语音通话所需信令的可靠转发与管理，支撑 P2P 语音通信全流程。
二、核心功能模块（细化说明）
（一）MQTT 客户端模块
MQTT 连接管理：与指定 MQTT 服务器建立 TCP 连接，支持用户名 / 密码认证、Keep-Alive 心跳保活机制；具备断连自动重连、网络波动恢复能力，实时记录连接状态日志。
主题订阅与消息接收：订阅 ESP32 设备上行信令主题（格式：接收"/voice/project/+/up"，下发 "/voice/project/%s/down"），支持通配符订阅多设备主题；解析 MQTT 消息体中 JSON 格式 WebRTC 信令，校验type/timestamp等必选字段的合法性。
消息转发：将 ESP32 设备上行信令按target字段（Web 端标识）精准转发至 WebSocket 信令服务器对应 Web 端会话。
消息发布：接收 WebSocket 服务器转发的 Web 端信令，按设备 ID 发布至 ESP32 下行 MQTT 主题（格式：/voice/project/%s/down）；采用 MQTT QoS 1 级别发布，确保信令可靠送达，处理发布确认反馈。
信令持久化：将接收到的所有 ESP32 设备上行信令，按时间戳完整写入项目目录下resources文件夹内的指定文件，留存原始信令数据用于后期调试。
（二）WebSocket 信令服务器模块
WebSocket 连接管理：监听指定端口（如 8080），接收 Web 浏览器 WebSocket 连接请求并建立双向通信通道；维护 “会话 ID-WebSocket 连接” 映射表，自动清理超时 / 断开的连接。
Web 端信令处理：接收 Web 端发送的 JSON 格式 WebRTC 信令，校验字段合法性；关联 Web 端会话 ID 与目标 ESP32 设备 ID，建立 “Web 端 - ESP32 设备” 唯一映射关系。
跨协议信令转发：将 Web 端信令转发至 MQTT 客户端，由其发布至 ESP32 下行主题；将 MQTT 客户端转发的 ESP32 信令推送至对应 Web 端 WebSocket 连接。
ICE 凭证管理：接收 ESP32 发送的ice_request信令，调用 STUN/TURN 服务器接口获取 ICE 凭证，封装为ice_response信令并通过 MQTT 下行主题推送至 ESP32。
通话状态管理：记录每路通话会话（Web 端 + ESP32 设备）的状态（呼叫中 / 通话中 / 已挂断）；处理bye/reject/timeout等异常信令，触发通话终止流程并清理会话映射关系。
（三）测试页面支持
在项目static目录下创建独立 HTML 测试页面，页面具备以下能力：
提供 WebSocket 连接配置项（服务器地址、端口），支持手动建立 / 断开与信令服务器的连接；
内置 WebRTC 语音通话核心逻辑，支持向指定 ESP32 设备发起offer信令、回复answer信令、交换candidate信令；
提供通话控制按钮（发起通话、接听、挂断、拒接），实时展示接收到的 ESP32 信令内容；
支持音频设备（麦克风 / 扬声器）选择与状态检测，满足用户直接通过该页面完成与 ESP32 设备的语音通话测试。
（四）WebRTC 信令流转全流程
ICE 凭证获取：ESP32 发送ice_request→MQTT 客户端转发→WebSocket 服务器获取凭证→封装ice_response→MQTT 客户端发布至 ESP32；
通话发起：Web/ESP32 发送offer信令→跨协议转发至目标端→目标端回复answer信令→双方交换candidate信令→建立 P2P 连接；
通话终止：任意端发送bye/reject/timeout信令→跨协议转发至对端→清理会话映射关系，终止语音传输。
三、核心特性
协议兼容：适配 MQTT 3.1.1（ESP32）、WebSocket RFC6455（Web 端），实现跨协议信令无缝转发；
信令标准化：所有信令统一采用 JSON 格式，核心字段（type/timestamp/target）规范统一，支持扩展customized自定义信令；
可靠性保障：MQTT QoS 1 级别发布、WebSocket 断线重连信令补发，避免关键信令丢失；
调试友好：ESP32 信令持久化存储、测试页面可视化操作，降低调试成本；
扩展性：支持多设备 / 多 Web 端并发会话，可扩展至视频通话等 WebRTC 场景。








附录，以下为esp32的mqtt信令格式
1. ICE 凭证请求 (ice_request)
   用途：设备向服务器请求 TURN/STUN ICE 服务器凭证，初始化和定时刷新时发送
   json
   {
   "type": "ice_request",
   "timestamp": 1740000000
   }
2. ICE 凭证响应 (ice_response)
   用途：服务器返回 TURN/STUN 凭证给设备（下行消息）
   json
   {
   "type": "ice_response",
   "ice_servers": [
   {
   "urls": ["turn:turn.example.com:3478"],
   "username": "user123",
   "credential": "pass456"
   }
   ]
   }
3. 呼叫发起 (offer)
   用途：呼出方（发起方）发送 SDP 信息，发起通话请求
   json
   {
   "target": "device_002",
   "timestamp": 1740000100,
   "type": "offer",
   "sdp": "v=0\r\no=- 12345 67890 IN IP4 192.168.1.1\r\ns=-\r\nt=0 0\r\n..."
   }
4. 呼叫应答 (answer)
   用途：呼入方（被叫方）回复 SDP 信息，确认接听通话
   json
   {
   "target": "device_001",
   "timestamp": 1740000200,
   "type": "answer",
   "sdp": "v=0\r\no=- 54321 09876 IN IP4 192.168.1.2\r\ns=-\r\nt=0 0\r\n..."
   }
5. ICE 候选信息 (candidate)
   用途：通话双方交换 ICE 网络候选地址，用于 P2P 连接建立
   json
   {
   "target": "device_002",
   "timestamp": 1740000300,
   "type": "candidate",
   "candidate": "candidate:1 1 UDP 2113937151 192.168.1.1 50048 typ host"
   }
6. 通话挂断 (bye)
   用途：通话任意一方主动挂断通话，结束会话
   json
   {
   "target": "device_001",
   "timestamp": 1740000400,
   "type": "bye"
   }
7. 呼叫拒接 (reject)
   用途：被叫方主动拒接来电
   json
   {
   "target": "device_001",
   "timestamp": 1740000500,
   "type": "reject"
   }
8. 呼叫超时 (timeout)
   用途：设备侧检测到呼叫超时，通知对方
   json
   {
   "target": "device_001",
   "timestamp": 1740000600,
   "type": "timeout"
   }
9. 自定义消息 (customized)
   用途：扩展自定义信令，传输业务相关的自定义数据
   json
   {
   "target": "device_002",
   "timestamp": 1740000700,
   "type": "customized",
   "data": "自定义业务数据内容"
   }
